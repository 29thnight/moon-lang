# PrSM 문서 작성 스타일 가이드

이 문서는 PrSM 프로젝트의 모든 문서(`docs/en/`, `docs/ko/`)에 적용되는 작성 규칙을 정의한다.

---

## 1. 파일 구조

### 1.1 위치

| 유형 | 경로 | 설명 |
|------|------|------|
| 영문 일반 문서 | `docs/en/*.md` | 가이드, 레퍼런스, 도구 |
| 한국어 일반 문서 | `docs/ko/*.md` | 영문과 1:1 대응 |
| 영문 언어 표준 | `docs/en/spec/*.md` | normative 사양 |
| 한국어 언어 표준 | `docs/ko/spec/*.md` | 영문과 1:1 대응 |
| 내비게이션 정의 | `docs/en/_nav.json`, `docs/ko/_nav.json` | 사이드바 메뉴 구조 |

### 1.2 새 문서 추가 절차

1. `docs/en/`에 `.md` 파일 생성
2. `docs/ko/`에 한국어 번역 파일 생성
3. `docs/en/_nav.json`과 `docs/ko/_nav.json`에 항목 추가
4. TypeScript 소스 수정 불필요 (네비게이션은 `_nav.json`에서 동적 로드)

### 1.3 Frontmatter

Jekyll 스타일 frontmatter를 포함하되, 실제 렌더링에는 사용되지 않는다 (React 앱이 strip). 향후 정적 사이트 생성기 전환을 위해 유지한다.

```yaml
---
title: 문서 제목
parent: 상위 카테고리
nav_order: 숫자
---
```

## 2. 마크다운 문법 (렌더러 지원 범위)

### 2.1 지원되는 문법

| 문법 | 예시 |
|------|------|
| h1 ~ h4 | `#`, `##`, `###`, `####` |
| 코드 블록 | ` ```lang ``` ` (prsm, csharp, cs, json, toml, ebnf) |
| 인라인 코드 | `` `code` `` |
| 볼드 | `**bold**` |
| 링크 | `[text](url)` |
| 테이블 | `\| col \| col \|` (이스케이프: `\\\|` → `\|`) |
| 비순서 리스트 | `- item` 또는 `* item` |
| 순서 리스트 | `1. item` |
| 수평선 | `---` |
| 블록인용 | `> text` |
| 버전 마커 | `(since PrSM 2)` 또는 `(PrSM 2 부터)` — 자동 초록색 |

### 2.2 지원되지 않는 문법 (사용 금지)

| 문법 | 대안 |
|------|------|
| h5 이상 (`#####`) | h4까지만 사용 |
| 중첩 리스트 | 평탄화하여 작성 |
| 이미지 `![alt](src)` | 미지원 |
| HTML 태그 | 미지원 |
| 각주 | 미지원 |
| `::: admonition :::` | 미지원 — `**텍스트**` 볼드로 대체 |
| 체크박스 `- [ ]` | 미지원 |

### 2.3 테이블에서 파이프 문자

테이블 셀 안에 `|` 문자가 필요한 경우 `\|`로 이스케이프한다.

```markdown
| 연산자 | 설명 |
|--------|------|
| `\|\|` | 논리 OR |
```

## 3. 코드 블록

### 3.1 지원 언어 태그

| 태그 | 하이라이팅 |
|------|-----------|
| `prsm`, `prism` | PrSM 전용 하이라이터 |
| `csharp`, `cs`, `c#` | C# 하이라이터 |
| `json` | JSON 하이라이터 |
| `toml`, `ebnf`, `text` | PrSM 하이라이터 폴백 |
| 기타 | 하이라이팅 없음 |

### 3.2 코드 예시 원칙

모든 언어 기능 설명에는 `.prsm` 소스와 생성 C# 쌍을 포함한다.

```markdown
` ` `prsm
val rb: Rigidbody = get()
` ` `

` ` `csharp
Rigidbody rb = GetComponent<Rigidbody>();
` ` `
```

## 4. 버전 마커

### 4.1 형식

| 언어 | 형식 | 렌더링 |
|------|------|--------|
| 영문 | `(since PrSM 2)` | 초록색 `#6aab73` 인라인 텍스트 |
| 한국어 | `(PrSM 2 부터)` | 초록색 `#6aab73` 인라인 텍스트 |

### 4.2 사용 위치

**인라인만 사용한다.** blockquote(`>`) 형식을 사용하지 않는다.

| 위치 | 예시 |
|------|------|
| 섹션 제목 | `### Listen lifetimes (since PrSM 2)` |
| 문서 제목 아래 | 제목 다음 줄에 `(since PrSM 2)` 단독 줄 |
| 본문 안 | `listen supports lifetime modifiers (since PrSM 2).` |
| 테이블 셀 | `\| New Input System sugar (since PrSM 2) \|` |

### 4.3 적용 대상

- v2에서 추가된 기능 전용 문서 → 문서 제목 아래에 마커
- 기존 문서 내 v2 추가 섹션 → 섹션 제목에 마커
- 언어 표준(standard.md) → 각 v2 섹션/규칙에 인라인 마커

### 4.4 금지 형식

다음 형식은 사용하지 않는다:
- `(v2)` ✗
- `> Added in v2.0` ✗
- `> [Since language 2]` ✗
- `v2.0에서 추가` ✗
- `::: warning` ✗

## 5. 제목 규칙

### 5.1 구분선

h1과 h2 제목 위에 자동으로 구분선이 삽입된다 (문서 첫 제목 제외). 마크다운에 `---`를 수동으로 넣을 필요 없다.

### 5.2 인라인 마크다운

모든 제목(h1~h4)에서 인라인 코드(`` `code` ``), 볼드(`**bold**`), 링크(`[text](url)`), 버전 마커가 렌더링된다.

### 5.3 앵커 태그

언어 표준에서 C++ 스타일 앵커 태그(`[scope]`, `[lex]` 등)를 제목에 포함할 수 있다. 렌더러는 이를 링크로 해석하지 않는다 (URL 부분이 없으므로).

## 7. 한/영 동기화 규칙

### 7.1 번역 원칙

- 모든 코드 블록은 번역하지 않는다
- 기술 용어(PrSM, C#, Unity, MonoBehaviour 등)는 영어 유지
- 에러 코드(E012, W001 등)는 영어 유지
- 버전 마커: 영문 `(since PrSM 2)` ↔ 한국어 `(PrSM 2 부터)`
- 산문, 제목, 테이블 헤더는 자연스러운 한국어로 번역

### 7.2 파일 대응

`docs/en/`의 모든 `.md` 파일은 `docs/ko/`에 대응 파일이 있어야 한다. `_nav.json`의 구조도 동일해야 한다.

## 8. 네비게이션 (`_nav.json`)

### 8.1 구조

```json
[
  {
    "id": "group-id",
    "title": "그룹 제목",
    "items": [
      { "id": "doc-id", "title": "문서 제목", "path": "docs/en/filename.md" }
    ]
  }
]
```

### 8.2 규칙

- `id`는 고유해야 한다 (전체 네비게이션에서 중복 불가)
- `path`는 프로젝트 루트 기준 상대 경로
- 하위 디렉토리 문서: `docs/en/spec/standard.md`
- 새 그룹 추가 시 en과 ko 모두에 동일 구조로 추가

### 8.3 문서 간 링크

문서 내에서 다른 문서를 참조할 때 `[text](filename.md)` 형식을 사용한다. 렌더러가 `.md` 링크를 가로채어 뷰어 내 페이지 전환으로 처리한다.

```markdown
See [Project Configuration](project-configuration.md) for details.
```

외부 URL은 일반 링크로 새 탭에서 열린다.

## 9. 언어 표준 문서 추가 규칙

### 9.1 normative 표현

- "shall" → 규범적 요구사항
- "shall not" → 규범적 금지
- "is defined as" → 정의

한국어: "해야 한다", "해서는 안 된다", "~로 정의된다"

### 9.2 에러 코드 참조

규칙 위반 시 발생하는 에러를 명시한다: "이 규칙을 위반하면 진단 **E082**가 발생한다."

### 9.3 C++ 스타일 앵커

섹션 제목에 `[tag]`를 포함한다: `## 3 Lexical structure [lex]`
