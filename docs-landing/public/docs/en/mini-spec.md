# Mini Spec

The compact implemented-language reference currently centers on:

- one top-level declaration per file
- `component`, `asset`, `class`, `data class`, `enum`, `attribute`
- null-safe types and generic collection references
- parenthesis-free control flow
- lifecycle blocks
- coroutine syntax and wait DSL
- `listen` event sugar
- `intrinsic` escape hatches
- `.prsmproject` based project scope

The source of truth for the compact language spec still lives in the repository planning docs, but the implemented surface is reflected throughout this documentation site.
